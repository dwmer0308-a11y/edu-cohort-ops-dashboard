# 飞书机器人通知配置说明

目标：当 CRM 登录过期或自动抓取失败时，系统能主动给你发飞书消息。

## 方式 A：使用已有自建应用

这是后续接飞书表格、审批时更统一的方式。

### 你在飞书开放平台里做

1. 打开飞书开放平台，进入你已经创建的企业自建应用。
2. 进入「凭证与基础信息」，找到 `App ID` 和 `App Secret`。
3. 进入「权限管理」，搜索并开通发送消息相关权限，优先开通「以应用的身份发消息」。
4. 如果平台提示需要发布或管理员审核，按提示发布到企业内可用。
5. 找到你的接收方式。最简单先用邮箱：`receive_id_type` 填 `email`，`receive_id` 填你的飞书账号邮箱。

### 你在本机里做

创建文件 `.local/feishu-notify.json`：

```json
{
  "mode": "app",
  "app_id": "这里填 App ID",
  "app_secret": "这里填 App Secret",
  "receive_id_type": "email",
  "receive_id": "这里填你的飞书账号邮箱"
}
```

然后运行测试：

```bash
node scripts/feishu_notify_test.mjs
```

如果你收到飞书消息，就说明配置成功。

## 方式 B：使用群机器人 Webhook

如果自建应用权限审核比较麻烦，可以先用群机器人 Webhook。

1. 在飞书群里添加自定义机器人。
2. 复制机器人 webhook 地址。
3. 创建 `.local/feishu-notify.json`：

```json
{
  "mode": "webhook",
  "webhook_url": "这里填群机器人 webhook"
}
```

然后运行：

```bash
node scripts/feishu_notify_test.mjs
```

## 注意

- 不要把 `App Secret` 或 webhook 发到聊天里。
- `.local/` 已经加入 `.gitignore`，只保存在你的电脑本地。
- 自建应用获取的 `tenant_access_token` 有有效期，脚本会每次发送前自动重新获取。
