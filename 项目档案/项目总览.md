# 营期转化监控看板

## 项目定位

这是一个面向书法项目营期经营的本地/局域网页看板，用于把目标配置、实际结果、营收推演和营期异常监控集中到一个系统里。

第一使用者：

- 项目负责人
- 转化负责人

核心要回答的问题：

- 今天 GMV / Leads 是否达成？
- 哪个营期转化异常？
- 哪个渠道或分类 ROI / R值偏差明显？
- 本月历史实际 + 未来规划后，预计能完成多少 GMV？

## 当前运行方式

本地访问：

```text
http://127.0.0.1:8765
```

局域网访问：

```text
http://本机局域网IP:8765
```

最近一次日志显示的局域网地址：

```text
http://10.8.55.209:8765
```

如果 IP 变化，以启动日志为准。

## 核心文件

- `server.py`：本地 HTTP 服务和数据接口。
- `public/index.html`：页面结构。
- `public/app.js`：核心业务逻辑和页面渲染。
- `public/styles.css`：UI 样式。
- `data/config.json`：目标、营期、实际导入、快照等配置数据。
- `data/backups/`：配置备份。
- `启动局域网服务器.command`：手动启动服务。
- `安装为局域网常驻服务.command`：安装/重启常驻服务。
- `安全清理缓存.command`：安全清理本机缓存的工具。

## 当前页面结构

- 总览：经营工作台，展示 Leads / GMV / 预估达成 / 异常提醒 / 月度进度。
- 计划中心：统一维护建期计划、R值目标、Leads预算。
- 营收日历：按自然日、营期、D阶段展示实际与推演。
- 营期监控：默认只看异常营期，可切换全部营期。
- 配置中心：维护渠道、子渠道、老师、接量规则、配置备份。

## 当前数据口径

目标侧：

- Leads 目标来自日历目标。
- GMV 目标来自营期 Leads × 对应 D4-D13 R值。
- R值目标按子渠道配置，可复制到营期。
- 同一开课日期存在多个小营期时，当前按小营期平分目标量级。

实际侧：

- 通过结果 CSV 导入实际营期数据。
- 以营期名称精确匹配规划营期。
- 未匹配规划的实际营期会作为历史导入营期展示。

阶段规则：

- 接量日期命中：接量期。
- D1-D3：行课期。
- D4-D6：转化期。
- D7-D13：追单期。
- D14 起：已封板。
- 接量前：待接量。

## 当前设计方向

UI 方向采用“清爽计划工作台”：

- 蓝绿为主。
- 页面减少重复入口。
- 总览只放经营负责人最需要看的信息。
- 计划、监控、推演分层展示。
- 卡片边角控制在 8px 内，保持工具感。
