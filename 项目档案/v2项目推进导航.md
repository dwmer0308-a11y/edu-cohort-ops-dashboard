# v2 项目推进导航

生成日期：2026-06-15

## 当前项目目标

把现有“书法项目管理看板”升级为一期聚焦 `书画/书法 + 朗诵` 的营期制教育业务经营系统。

当前不做全 SKU 展开，不直接推倒重做，不立刻进入 Loop 自动实现。

## 我的引导职责

后续你不需要记住什么时候用什么技能。

我会在每一步主动告诉你：

- 当前该进入哪个阶段
- 是否需要新开线程
- 该点名哪个技能
- 这一步要产出什么
- 产出应该放在哪里
- 是否可以进入下一步

你只需要做两件事：

- 判断业务方向对不对
- 在我提示时确认继续、调整或暂停

## 总路线

### 阶段 A：业务规划收口

目标：把一期范围、业务对象、数据口径、页面边界定住。

当前状态：基本完成，需要最后收口。

核心文档：

- `项目档案/营期制教育业务经营系统规划.md`
- `项目档案/v2项目推进导航.md`

阶段 A 完成标准：

- 一期 SKU 确认为 `书画/书法 + 朗诵`
- 页面范围确认
- 数据源清单确认
- 暂不做的内容写清楚

建议使用技能：

- `/plan-ceo-review`：用于战略和范围判断

你需要输入的话：

```text
用 /plan-ceo-review 帮我把一期范围收口，确认哪些做、哪些不做。
```

### 阶段 B：设计系统和页面原型

目标：先看见系统长什么样，再决定怎么开发。

核心产出：

- 设计系统：颜色、字体、布局、表格、标签、异常状态
- 经营总览原型
- SKU 作战室原型
- 课程分析页原型

建议使用技能顺序：

1. `/design-consultation`
2. `/design-shotgun`
3. `/plan-design-review`

你需要输入的话：

```text
用 /design-consultation 给这个 v2 经营系统定设计系统。
```

之后我会提示你是否继续：

```text
用 /design-shotgun 给经营总览、SKU 作战室、课程分析页各出几个方向。
```

阶段 B 完成标准：

- 你选定一个页面风格方向
- 三个核心页面的信息层级确定
- 不再靠文字想象页面

### 阶段 C：数据资产和接口方案

目标：把 Metabase、订单、成本、课程数据变成系统可用的数据层规划。

核心产出：

- 数据源字典
- SKU 映射
- 营期主数据方案
- 快照/缓存策略
- 字段校验规则

建议使用技能：

- `/plan-eng-review`：用于数据架构和工程方案审核

你需要输入的话：

```text
用 /plan-eng-review 帮我审一下 v2 数据资产层和快照方案。
```

阶段 C 完成标准：

- 明确哪些数据实时拉
- 哪些数据封板后固定
- 哪些数据只存摘要
- 哪些接口可能截断
- 书画/书法映射进入主数据

### 阶段 D：第一批工程实现

目标：在旧系统旁边做 v2 第一批可用页面。

建议先实现：

- v2 经营总览
- v2 SKU 作战室
- 基础数据资产状态

暂不先实现：

- 全部课程视频复盘
- 全 SKU 扩展
- 自动 Loop
- 复杂权限系统

建议使用技能：

- `/plan-eng-review`
- `/qa`
- `/design-review`

阶段 D 完成标准：

- 本地可打开 v2 页面
- 数据能从本地缓存/样本驱动
- 页面有 loading、empty、error 状态
- 不影响旧系统使用

### 阶段 E：接 GitHub 和 Loop

目标：当方向稳定后，把工程协作和自动化做起来。

先接 GitHub，再考虑 Loop。

适合 Loop 的任务：

- 批量补页面状态
- 批量接字段
- 批量做表格
- 批量跑 QA
- 修复重复视觉问题

不适合 Loop 的任务：

- 定义业务口径
- 判断经营指标
- 决定页面优先级
- 处理模糊需求

阶段 E 完成标准：

- GitHub 仓库已配置
- 分支/PR 流程可用
- 有明确 PRD
- 有测试和回滚策略
